1️⃣ What Is JSON Schema?

JSON Schema is a contract/specification that defines:
Structure of JSON
Mandatory vs optional fields
Data types
Allowed values
Field formats & constraints


JSON Schema vs Normal Assertions
Aspect						Assertions				JSON Schema
Structure validation		❌ Manual				✅ Automatic
Data type validation		❌ Multiple asserts		✅ Built-in
Mandatory fields			❌ Manual				✅ Required
Extra fields				❌ Hard					✅ Easy
Maintenance					❌ High					✅ Low


9️⃣ Best Practices (Senior-Level Answer)

Maintain separate schema files
One schema per endpoint & response type
Validate both success & error responses
Use additionalProperties: false
Version schemas along with APIs
Run schema validation in CI pipelines

“JSON Schema defines the API contract by validating structure, mandatory fields, data types, and allowed values, ensuring backward compatibility and preventing breaking changes.”
