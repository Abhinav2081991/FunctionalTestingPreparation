Serenity BDD – Overview for a Senior QA Specialist
What is Serenity BDD?

Serenity BDD is a mature open-source test automation framework built on top of:

Selenium WebDriver (for UI)
Cucumber (for BDD)
JUnit / TestNG
Rest Assured (for API automation)

It is designed to:

Improve test maintainability
Provide rich living documentation
Bridge business and technical teams
Reduce flakiness via built-in wait strategies
Generate advanced reporting automatically

Serenity directly addresses these.


🧱 Core Architectural Concepts
1️⃣ Page Objects (Enhanced)

Serenity extends Page Object Model with:

Built-in smart waits
Automatic step reporting
WebElementFacade instead of raw WebElement

Instead of:

driver.findElement(...)

You use:

@FindBy(id="login")
WebElementFacade loginButton;

WebElementFacade:

Waits automatically
Provides rich logging
Reduces explicit wait clutter

2️⃣ Step Libraries (Business Layer)

This is the real power.

You don’t write tests like Selenium scripts.
You write business steps:

@Step("User logs in with valid credentials")
public void login(String user, String pass) {
    loginPage.enterUsername(user);
    loginPage.enterPassword(pass);
    loginPage.clickLogin();
}

Now reports show:

✔ User logs in with valid credentials
Instead of technical noise.

This improves:

Stakeholder readability
Debug traceability
Living documentation

3️⃣ Screenplay Pattern (Enterprise-Grade Design)

For large automation programs, Serenity promotes Screenplay pattern.

Instead of:

Test → Step → Page

You use:

Actor → Tasks → Interactions → Questions

Example:

Actor abhinav = Actor.named("Abhinav");

abhinav.attemptsTo(
    Login.withCredentials("admin","pass")
);

Benefits:

High reusability
Low coupling
Better scalability for microservices/UI/API hybrid systems
Easier parallel execution

For enterprise frameworks, Screenplay is superior to classic POM.

📦 2️⃣ Maven pom.xml
<project>
    <modelVersion>4.0.0</modelVersion>

    <groupId>com.company.automation</groupId>
    <artifactId>serenity-framework</artifactId>
    <version>1.0</version>

    <properties>
        <serenity.version>3.9.8</serenity.version>
        <maven.compiler.source>17</maven.compiler.source>
        <maven.compiler.target>17</maven.compiler.target>
    </properties>

    <dependencies>
        <!-- Serenity Core -->
        <dependency>
            <groupId>net.serenity-bdd</groupId>
            <artifactId>serenity-core</artifactId>
            <version>${serenity.version}</version>
        </dependency>

        <!-- Serenity JUnit -->
        <dependency>
            <groupId>net.serenity-bdd</groupId>
            <artifactId>serenity-junit</artifactId>
            <version>${serenity.version}</version>
        </dependency>

        <!-- WebDriver -->
        <dependency>
            <groupId>org.seleniumhq.selenium</groupId>
            <artifactId>selenium-java</artifactId>
            <version>4.18.1</version>
        </dependency>
    </dependencies>

    <build>
        <plugins>
            <plugin>
                <groupId>net.serenity-bdd.maven.plugins</groupId>
                <artifactId>serenity-maven-plugin</artifactId>
                <version>${serenity.version}</version>
                <executions>
                    <execution>
                        <id>serenity-reports</id>
                        <phase>post-integration-test</phase>
                        <goals>
                            <goal>aggregate</goal>
                        </goals>
                    </execution>
                </executions>
            </plugin>
        </plugins>
    </build>
</project>

🧑‍💻 5️⃣ Page Object (LoginPage.java)
package pages;

import net.serenitybdd.core.pages.PageObject;
import net.serenitybdd.core.annotations.findby.FindBy;
import net.serenitybdd.core.pages.WebElementFacade;

public class LoginPage extends PageObject {

    @FindBy(id = "username")
    WebElementFacade usernameField;

    @FindBy(id = "password")
    WebElementFacade passwordField;

    @FindBy(id = "loginBtn")
    WebElementFacade loginButton;

    public void enterUsername(String username) {
        usernameField.type(username);
    }

    public void enterPassword(String password) {
        passwordField.type(password);
    }

    public void clickLogin() {
        loginButton.click();
    }
}
✅ Why WebElementFacade?

Built-in waits
Auto synchronization
Better reporting
Less flaky than raw WebElement

🧠 6️⃣ Step Library (LoginSteps.java)

This is where Serenity shines.

package steps;

import net.thucydides.core.annotations.Step;
import pages.LoginPage;

public class LoginSteps {

    LoginPage loginPage;

    @Step("User opens login page")
    public void openLoginPage() {
        loginPage.open();
    }

    @Step("User logs in with username {0} and password {1}")
    public void login(String username, String password) {
        loginPage.enterUsername(username);
        loginPage.enterPassword(password);
        loginPage.clickLogin();
    }
}

🔹 The @Step annotation ensures:

Each method appears in Serenity report
Business readable reporting
Screenshot capture per step

🧪 7️⃣ Test Class (LoginTest.java)
package tests;

import net.serenitybdd.junit.runners.SerenityRunner;
import net.thucydides.core.annotations.Steps;
import org.junit.Test;
import org.junit.runner.RunWith;
import steps.LoginSteps;

@RunWith(SerenityRunner.class)
public class LoginTest {

    @Steps
    LoginSteps loginSteps;

    @Test
    public void user_should_login_successfully() {
        loginSteps.openLoginPage();
        loginSteps.login("admin", "password123");
    }
}

🧩 1️⃣0️⃣ Adding API Test (Serenity + Rest Assured)

Add dependency:

<dependency>
  <groupId>net.serenity-bdd</groupId>
  <artifactId>serenity-rest-assured</artifactId>
  <version>${serenity.version}</version>
</dependency>

Example:

import net.serenitybdd.rest.SerenityRest;
import org.junit.Test;

public class LoginApiTest {

    @Test
    public void should_login_via_api() {

        SerenityRest.given()
            .header("Content-Type", "application/json")
            .body("{\"username\":\"admin\",\"password\":\"password123\"}")
            .post("/api/login")
            .then()
            .statusCode(200);
    }
}

Automatically logs:

Request
Response
Headers
Status code
Payload

All inside Serenity report.
