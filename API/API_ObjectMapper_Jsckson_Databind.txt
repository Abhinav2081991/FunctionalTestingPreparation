Step 1 — What Happens When You Call .body(requestBody)

When you pass:

.body(requestBody)


You are giving Rest Assured a Java object, not a JSON string.
Rest Assured checks:

What is the object type?
What is the Content-Type header?
What object mappers are available on the classpath?

🔷 Step 2 — Content-Type Triggers Serialization

You set:

.contentType("application/json")


This tells Rest Assured:

“I need to convert this Java object into JSON before sending it.”

Now Rest Assured searches for a JSON serializer.

🔷 Step 3 — How It Finds a Serializer

Internally, Rest Assured looks for JSON libraries in your project:

Jackson (most common)
Gson
Johnzon
JSON-B

If Jackson is present in your dependencies (which most frameworks use), Rest Assured uses:

com.fasterxml.jackson.databind.ObjectMapper


Jackson performs the actual conversion.

🔷 Step 4 — What Jackson Does Internally

Let’s say you have:

@Data
@Builder
public class UserRequest {
    private String name;
    private int age;
    private String role;
}


And you create:

UserRequest requestBody = UserRequest.builder()
        .name("Abhinav")
        .age(30)
        .role("ADMIN")
        .build();


Jackson performs:

Reflection-Based Inspection

It:

Reads all fields
Uses getters (generated by Lombok)
Converts fields into JSON keys
Maps values accordingly

It effectively does:

ObjectMapper mapper = new ObjectMapper();
String json = mapper.writeValueAsString(requestBody);


Resulting JSON:

{
  "name": "Abhinav",
  "age": 30,
  "role": "ADMIN"
}


Rest Assured sends this JSON as HTTP body.

🔷 Step 5 — Why Lombok Works Perfectly Here

Because Lombok generates:

Getters
Setters
Constructor
Builder
Jackson relies on getters/setters or fields via reflection.

So Lombok + Jackson integrate seamlessly.

🔷 Important Internal Flow (Architect View)

When .body(requestBody) is called:

Rest Assured detects non-String object

It checks Content-Type

It invokes ObjectMapper

Object → JSON String

JSON added to HTTP request body

HTTP client sends request

All automatic.

🔷 How to Confirm Serialization Is Working

You can enable logging:

given()
    .log().body()
    .contentType("application/json")
    .body(requestBody)
.when()
    .post("/users");


You’ll see the serialized JSON printed in console.

🔷 What If No JSON Library Is Present?

If Jackson/Gson is NOT on classpath:

You get an error like:

Cannot serialize because no JSON serializer found


That’s why you must include:

<dependency>
  <groupId>com.fasterxml.jackson.core</groupId>
  <artifactId>jackson-databind</artifactId>
</dependency>

🔷 Customizing Serialization (Advanced)

You can configure custom ObjectMapper:

RestAssured.config = RestAssured.config()
    .objectMapperConfig(objectMapperConfig()
        .jackson2ObjectMapperFactory((cls, charset) -> {
            ObjectMapper mapper = new ObjectMapper();
            mapper.findAndRegisterModules();
            return mapper;
        }));


Used when:

Handling dates

Custom naming strategies

Ignoring nulls

Special formatting

🔷 Reverse Process — Deserialization

When you do:

.extract().as(UserResponse.class);


Jackson performs:

JSON → Java Object


Using:

mapper.readValue(json, UserResponse.class);

✅ Scenario 1 — Read JSON File as String and Send Directly
📂 Example JSON file: user.json
{
  "name": "Abhinav",
  "age": 30,
  "role": "ADMIN"
}

✅ Step 1 — Read JSON as String
import java.nio.file.Files;
import java.nio.file.Paths;
import java.io.IOException;

public class JsonUtil {

    public static String readJson(String filePath) throws IOException {
        return new String(Files.readAllBytes(Paths.get(filePath)));
    }
}

✅ Step 2 — Send as Request Body
import static io.restassured.RestAssured.*;

String jsonBody = JsonUtil.readJson("src/test/resources/user.json");

given()
    .baseUri("https://api.example.com")
    .contentType("application/json")
    .body(jsonBody)
.when()
    .post("/users")
.then()
    .statusCode(201);


✔ This sends raw JSON string.

✅ Scenario 2 — Read JSON → Convert to POJO → Modify → Send

This is the enterprise approach.

Step 1 — Create POJO
import lombok.Data;

@Data
public class UserRequest {
    private String name;
    private int age;
    private String role;
}

Step 2 — Read JSON and Convert to Object
import com.fasterxml.jackson.databind.ObjectMapper;

ObjectMapper mapper = new ObjectMapper();

String json = JsonUtil.readJson("src/test/resources/user.json");

UserRequest request = mapper.readValue(json, UserRequest.class);


Now you have a Java object.

Step 3 — Modify Dynamically (Very Powerful)
request.setName("DynamicName_" + System.currentTimeMillis());


This is why enterprise teams prefer ObjectMapper.

Step 4 — Convert Back to JSON (Optional)
String updatedJson = mapper.writeValueAsString(request);

Step 5 — Send Request
given()
    .baseUri("https://api.example.com")
    .contentType("application/json")
    .body(updatedJson)
.when()
    .post("/users")
.then()
    .statusCode(201);


OR simply:

.body(request)


Rest Assured will serialize automatically.

✅ Scenario 3 — Using ObjectMapper Directly Without String

Cleanest version:

ObjectMapper mapper = new ObjectMapper();

UserRequest request =
        mapper.readValue(
                new File("src/test/resources/user.json"),
                UserRequest.class
        );

given()
    .baseUri("https://api.example.com")
    .contentType("application/json")
    .body(request)
.when()
    .post("/users")
.then()
    .statusCode(201);


No manual String handling needed.
