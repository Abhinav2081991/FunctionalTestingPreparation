5️⃣ Step Definition – Lombok + Builder Pattern
@Given("user provides multiple user details")
public void user_provides_multiple_user_details(DataTable dataTable) {

    List<Map<String, String>> rows = dataTable.asMaps();

    List<User> users = rows.stream()
            .map(row -> User.builder()
                    .name(row.get("name"))
                    .role(row.get("role"))
                    .address(Address.builder()
                            .city(row.get("city"))
                            .zip(row.get("zip"))
                            .country(row.get("country"))
                            .build())
                    .build())
            .toList();

    UserRequest request = UserRequest.builder()
            .users(users)
            .build();

    response = given()
            .contentType(ContentType.JSON)
            .body(request)
            .post("/users/bulk");
}

7️⃣ Why Builder Pattern is Preferred (Interview Trap)

❌ Constructors break when fields increase
❌ Setters allow invalid states

✔ Builder:

Readable
Order-independent
Safe for optional fields
Perfect for JSON payloads

❌ IDE annotation processing disabled

Lombok Annotations – Significance & Sequence
1️⃣ @Data
🔹 What it Does

Generates all of the following automatically:

Getters for all fields
Setters for non-final fields
toString()
equals() & hashCode()

RequiredArgsConstructor

@Data
public class User {
    private String name;
    private String role;
}

🔹 When to Use

✔ DTOs / POJOs
✔ API payload objects
✔ Cucumber DataTable mapping

⚠️ Caution

Not recommended for entities or immutable objects

Generates setters → mutable state

2️⃣ @Getter / @Setter
🔹 What they Do

Generate getters/setters individually.

@Getter
@Setter
public class User {
    private String name;
}

🔹 When to Use

✔ When you want control
✔ Avoid full @Data

3️⃣ @NoArgsConstructor
🔹 What it Does

Creates a no-argument constructor.

@NoArgsConstructor
public class User {}

🔹 Why Important

✔ Required by:

Cucumber DataTable

Jackson (JSON serialization/deserialization)

Reflection-based frameworks

4️⃣ @AllArgsConstructor
🔹 What it Does

Creates a constructor with all fields as parameters.

@AllArgsConstructor
public class User {
    private String name;
    private String role;
}

🔹 When to Use

✔ Manual object creation
✔ Test data setup

5️⃣ @Builder ⭐ (Most Important)
🔹 What it Does

Generates Builder pattern.

@Builder
public class User {
    private String name;
    private String role;
}

🔹 Why We Use It

✔ Clean object creation
✔ Avoid constructor overloads
✔ Order-independent
✔ Perfect for nested JSON

User.builder()
    .name("John")
    .role("Admin")
    .build();

6️⃣ @Value (Immutable Objects)
🔹 What it Does

Makes class final
Fields private & final
No setters
Generates getters, equals, hashCode, toString

@Value
@Builder
public class User {
    String name;
    String role;
}

🔹 When to Use

✔ Thread-safe objects
✔ Parallel execution
✔ Read-only data

7️⃣ @EqualsAndHashCode
🔹 What it Does

Generates equals() and hashCode().

@EqualsAndHashCode
public class User {}


✔ Useful for collections & comparisons

8️⃣ @ToString
🔹 What it Does

Generates toString().

@ToString
public class User {}


✔ Helpful for logging/debugging

9️⃣ @Builder.Default
🔹 What it Does

Sets default value when using Builder.

@Builder.Default
private String role = "User";


✔ Without this, builder ignores default values

🔢 Do Lombok Annotations Have to Be in Sequence?
❌ NO — Order Does NOT Matter
@Data
@Builder
@NoArgsConstructor
@AllArgsConstructor
public class User {}


⬇️ Same as ⬇️

@Builder
@AllArgsConstructor
@NoArgsConstructor
@Data
public class User {}

📌 Why?

Lombok is annotation-driven

Java compiler processes them independently

🚨 But There ARE Some Rules (Interview Gold)
❗ Rule 1: @Builder + Constructors

If you add custom constructors, builder may break

Prefer:

@Builder
@AllArgsConstructor
@NoArgsConstructor

❗ Rule 2: @Value vs @Data
@Data	@Value
Mutable	Immutable
Setters	No setters
Not thread-safe	Thread-safe

❌ Do NOT use together

❗ Rule 3: Framework Compatibility

Framework				Required
Cucumber				@NoArgsConstructor
Jackson					@NoArgsConstructor
REST APIs				@Builder
Parallel tests			@Value


🧠 Interview One-Liner (Very Strong)

Lombok annotations remove boilerplate code by generating constructors, getters, setters, and builders at compile time. The order of annotations does not matter, but choosing the right combination is critical for framework compatibility and thread safety.

✅ Recommended Combination (Automation Framework)
@Data
@Builder
@NoArgsConstructor
@AllArgsConstructor
public class User {}


🔥 Clean
🔥 Flexible
