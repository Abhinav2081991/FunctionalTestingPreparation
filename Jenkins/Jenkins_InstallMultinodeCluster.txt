For the Jenkins Controller (Master)
The method for setting the JVM options for the controller depends on how Jenkins was installed and which service manager it uses (e.g., systemd, init, or a Windows service). 
Locate the configuration file/method:
Linux (Systemd): Use systemctl edit jenkins to create an override file and add the Java options.
Linux (Init/Ubuntu prior to March 2022 releases): Edit the JAVA_ARGS or JAVA_OPTS line in the /etc/default/jenkins file.
Windows: Modify the jenkins.xml file in your installation directory and add the -Xmx parameter to the <arguments> tag.
Docker: Use environment variables like JAVA_MAX_HEAP_PARAM or CONTAINER_HEAP_PERCENT when launching the container to control the heap size.
Add/Modify the JVM options:
Specify the initial (-Xms) and maximum (-Xmx) heap sizes. Best practice is to set them to the same value to avoid dynamic resizing issues.
Example: -Xms4g -Xmx4g -XX:+UseG1GC (sets initial and max heap to 4GB and uses the G1 Garbage Collector).
Restart Jenkins: After saving the changes, restart the Jenkins service for the new settings to take effect.

For a Jenkins Agent Node
You can configure the JVM options for agents directly within the Jenkins UI or via environment variables if using containers. 
Navigate to the Node configuration:
From the Jenkins dashboard, go to Manage Jenkins > Manage Nodes and Clouds.
Select the specific agent you want to configure and click Configure.
Add JVM options in the UI:
In the Node configuration page, find the Launch method section.
Click the Advanced button.
Add the desired JVM arguments (e.g., -Xms2g -Xmx2g) in the JVM Options field.
Save and Relaunch:
Save the configuration.
If the agent is already online, you may need to Relaunch agent to apply the new settings.
