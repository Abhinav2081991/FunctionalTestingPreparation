🥇 Option 1: Jenkins Credentials Store (BEST PRACTICE)


UI -  Jenkins  --> Manage Jenkins  --> Credentials  --> System  -- Add Credentials.



Jenkins has a secure credential vault.

Supported credential types:
Type						Use Case
Secret File					Keystore / Truststore
Secret Text					Passwords, tokens
Username & Password			Basic auth
Certificate					Client cert auth

--------------------------------------------------------------------------------------------

Step-by-step: Store keystore/truststore
1️⃣ Jenkins UI
Manage Jenkins → Credentials → Global → Add Credentials

2️⃣ Choose:
Kind: Secret file

3️⃣ Upload:

client-keystore.jks

truststore.jks

Give IDs:

client_keystore
client_truststore

🔹 Use in Jenkinsfile

withCredentials([
    file(credentialsId: 'client_keystore', variable: 'KEYSTORE_FILE'),
    file(credentialsId: 'client_truststore', variable: 'TRUSTSTORE_FILE'),
    string(credentialsId: 'keystore_password', variable: 'KEYSTORE_PASS')
]) {
    bat """
    mvn test ^
      -Djavax.net.ssl.keyStore=%KEYSTORE_FILE% ^
      -Djavax.net.ssl.keyStorePassword=%KEYSTORE_PASS% ^
      -Djavax.net.ssl.trustStore=%TRUSTSTORE_FILE%
    """
}


✔ Files exist only during build
✔ Deleted automatically
✔ Fully masked in logs

🔥 Interview-ready line:

“In Jenkins, keystores and truststores are stored as Secret File credentials, injected at runtime and never persisted on disk.”
